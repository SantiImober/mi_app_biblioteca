Notas de Diseño

Tabla Prestamos como pivote: actúa como tabla intermedia entre Usuarios y Libros, implementando una relación N:M descompuesta en dos relaciones 1:N.

Estado del préstamo: se determina por fecha_devolucion. Si es NULL, el préstamo está activo. Si tiene valor, está cerrado. Esta decisión evita un campo booleano y agrega información temporal.

Stock en tiempo real: cantidad_ejemplares se decrementa al prestar y se incrementa al devolver, manteniendo el stock actualizado automáticamente.
